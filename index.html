Introduction

This guide provides information on integrating external systems with the ATS, including connections, retrieving job listings, and submitting applications.

Authentication

Authentication uses Salesforce Connected App and OAuth 2.0 flow. After getting the Access Token, use it for API requests.

Authorization: Bearer <Access_Token>

API Information

Base URL: https://<your-instance>.salesforce.com/services/data/vXX.0/

Use latest Salesforce API version (e.g., v60.0 for Summer '24).

Common Request Headers:

  • Authorization: Bearer Token
  • Content-Type: application/json
  • Accept: */*

Record Operations

Get Records

GET /services/data/vXX.0/query/?q=SELECT+Name+FROM+bpats__Job__c

Create Record

POST /services/data/vXX.0/sobjects/bpats__Job__c
{
  "Name": "New Job"
}

Bulk Create Records

POST /services/data/vXX.0/composite/tree/bpats__Job__c

Update Record

PATCH /services/data/vXX.0/sobjects/bpats__Job__c/{recordId}

Delete Record

DELETE /services/data/vXX.0/sobjects/bpats__Job__c/{recordId}

Permission Sets

Different permission sets are provided based on the external system access required, such as:

  • ATS Job Portal Permission Set
  • External Interview Feedback Permission Set
  • ATS User Permission Set
  • ATS Admin Permission Set

Error Handling

Common API Response Codes:

  • 200 OK - Successful GET, PATCH
  • 201 Created - Successful POST
  • 400 Bad Request - Invalid request body
  • 401 Unauthorized - Invalid or expired token
  • 404 Not Found - Resource not found

API Limits

Salesforce API usage depends on your Edition and licenses. Developer Edition: 15,000/day, Enterprise Edition: 100,000/day + user license multiplier.

Use GET /services/data/vXX.0/limits to check your current limits programmatically.
